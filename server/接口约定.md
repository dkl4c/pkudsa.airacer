# 此文档描述了与 SimNode 部分交互的接口约定

根据对战平台的显示需求，可能需要的数据如下：

## 这部分需要使用 API 实时调用，为动态数据
1. 实时的赛场画面（根据文档需求，不一定有这方面的需求）
    1. 赛场总俯视图
    2. 某一车辆的视觉画面
    3. 某一车辆当前运行状态：空间位置、速度、转向角度等
    4. 实时排名（根据车辆位置计算）
      p.s. 车辆实时排名等逻辑应该由 SimNode 部分负责还是后端处理？个人认为 SimNode 可以获取第一手的仿真数据，由 SimNode 处理更加方便

## 这部分在仿真结束后，生成静态数据，独立加载
2. 仿真完成后的回放数据，由 SimNode 生成，后端加载显示

3. 仿真完成后的统计数据
    1. 车辆最终排名信息
    2. 各个车辆行驶总时间
    3. 各个车辆获取道具数量
    4. 各个车辆碰撞次数
    5. ...根据文档添加
